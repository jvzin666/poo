POO - Definições
- Classe (concreta ou abstrata)
Classe concreta: é uma classe normal, que pode gerar objetos.

Classe abstrata: é uma classe que serve como base para outras. Ela não pode ser instanciada e pode ter métodos sem implementação, que obrigam as subclasses a implementá-los.

- objeto
O objeto é a representação real da classe em funcionamento. Ele possui valores próprios para seus atributos e pode executar os métodos definidos pela classe.

- Instancia
Instância é o processo de criar um objeto a partir de uma classe. Quando um objeto é criado, dizemos que ele é uma instância daquela classe.

- atributo
Os atributos são as características de uma classe ou objeto. Eles armazenam informações que descrevem o estado do objeto.

- constante
Uma constante é um valor que não pode ser alterado após ser definido. É usada para representar valores fixos que não devem mudar durante a execução do programa.

- interface
A interface define um conjunto de métodos que uma classe deve implementar. Ela não possui implementação, apenas define o que deve ser feito, funcionando como um contrato. 

- modificadores de acesso
Os modificadores de acesso controlam a visibilidade de classes, atributos e métodos, protegendo os dados do sistema.

public: pode ser acessado por qualquer classe.

private: pode ser acessado apenas dentro da própria classe.

protected: pode ser acessado pela classe e suas subclasses.

padrão (default): acessível apenas por classes do mesmo pacote.










pilares do poo




abstração
A abstração consiste em mostrar apenas as informações essenciais de um objeto, ocultando detalhes desnecessários para quem o utiliza.

herança
A herança permite que uma classe reutilize atributos e métodos de outra classe, criando uma relação de hierarquia entre elas.

encapsulamento
O encapsulamento protege os dados do objeto, impedindo acesso direto aos atributos e permitindo que sejam manipulados apenas por métodos controlados.

polimorfismo
O polimorfismo permite que métodos com o mesmo nome tenham comportamentos diferentes, dependendo da classe do objeto que os executa.
